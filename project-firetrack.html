FireTrack — Hybrid Indoor Firefighter Localization & Health Monitor

A wearable embedded system that tracks firefighters inside burning buildings using dead reckoning and RFID-corrected positioning, while monitoring vital signs in real-time through a command center dashboard.

Project Overview

When firefighters enter a burning building, GPS signals are blocked. Incident commanders on the outside have no way to know where their crew members are, whether they're moving, or if they need medical attention.

FireTrack solves this by combining IMU-based dead reckoning with RFID anchor correction and Kalman filter sensor fusion to provide reliable indoor positioning — even in GPS-denied, smoke-filled environments. A real-time WiFi dashboard displays each firefighter's location, path history, and vital signs.

The Problem

Indoor firefighter safety is a critical challenge with multiple failure points:

  • GPS does not work inside buildings — incident commanders are blind
  • Radio communication can be unreliable in multi-story structures
  • No real-time health monitoring for heat stress or cardiac events
  • Dead reckoning alone drifts over time without correction points
  • Existing solutions require expensive infrastructure changes

System Architecture

The system operates in three layers — each handling a specific aspect of tracking and monitoring:

Sensing Layer

Boot-mounted IMU (MPU9250) detects footsteps and heading via accelerometer + gyroscope. RFID reader scans passive anchor tags at known positions. MAX30102 and LM35 sensors monitor heart rate and body temperature.

Processing & Localization Layer

The VEGA RISC-V processor runs step detection, heading estimation (dead reckoning), and applies a Kalman filter to fuse IMU data with RFID anchor positions — correcting drift errors in real-time.

Communication & Dashboard Layer

ESP8266 WiFi module creates a local access point and web server. The command center dashboard displays a floor map with live position, path history, vital signs, and emergency alerts (e.g., "MAN DOWN — NO MOVEMENT").

Key Code — Dead Reckoning & RFID Correction

The core navigation logic uses step detection from the IMU to update position via dead reckoning, then corrects drift when an RFID anchor is scanned:

C / Arduino main.ino — Core Navigation Loop 
void loop()
{
    readIMU();

    // Step Detection → Dead Reckoning
    if(detectStep())
    {
        heading = calculateHeading();
        // Update position using trigonometry
        posX = posX + stepLength * cos(heading);
        posY = posY + stepLength * sin(heading);
    }

    // RFID Anchor → Correct accumulated drift
    rfidTag = readRFID();
    if(rfidTag != "")
    {
        correctPosition(rfidTag);  // Snap to known anchor coords
    }

    sendToDashboard();  // Push to WiFi command center
    delay(200);
}

The hybrid approach is key — dead reckoning provides continuous tracking between anchors, while RFID tags at known positions reset accumulated drift errors. The Kalman filter smoothly blends both data sources for stable position estimates.

How Tracking Works

🥾

1. Step Detection

The boot-mounted IMU detects each footstep through acceleration peaks. When a step is detected, the heading (direction) is calculated from the gyroscope/compass data.

📍

2. Dead Reckoning

Using the step length (~0.7m) and heading angle, the system calculates the new X/Y position using trigonometry. This runs continuously even without any infrastructure.

🏷️

3. RFID Correction

Passive RFID tags pre-installed at known locations (doorways, stairwells) are scanned automatically. When detected, the system snaps position to the known anchor coordinates.

🔄

4. Kalman Fusion

A Kalman filter smoothly blends the continuous dead reckoning data with discrete RFID corrections, producing stable position estimates with bounded error.

Command Center Dashboard

A web-based dashboard served locally over WiFi, showing real-time firefighter position, vital signs, and emergency alerts.

🗺️

Live Floor Map

Shows the firefighter's real-time position on a building floor plan with path history trace. Updates every 200ms for smooth tracking.

❤️

Vital Signs Monitor

Displays heart rate (BPM), SpO2, and body temperature. Alerts if vitals exceed safe thresholds during fire operations.

🚨

Emergency Alerts

"MAN DOWN — NO MOVEMENT" alert triggers automatically if no motion is detected for a configurable duration, indicating a downed firefighter.

📊

Navigation Data

Shows heading, position coordinates, step count, distance traveled, and floor number — giving commanders full situational awareness.
