Project Overview

This project addresses the challenge of inefficient water usage in agriculture by building a smart irrigation controller. The system uses a Raspberry Pi Pico W as its brain, interfacing with capacitive soil moisture sensors, a weather forecast API, a 4x4 matrix keypad, and an I2C LCD display to make intelligent irrigation decisions.

Unlike simple timer-based systems, this controller evaluates real-time soil conditions and upcoming weather forecasts before deciding whether to irrigate — preventing unnecessary water usage when rain is expected.

The Problem

Traditional irrigation systems either rely on fixed schedules or manual intervention, leading to:

  • Overwatering and water wastage when soil is already moist
  • Irrigating right before rainfall — wasting water and electricity
  • Lack of real-time monitoring for farmers in remote locations
  • No intelligent decision-making based on environmental conditions

How It Works

The system operates in multiple modes selectable via the keypad interface:

1

Soil Moisture Mode

Continuously monitors soil moisture using dual capacitive sensors. Automatically turns the pump ON when soil is dry and OFF when wet. Real-time values displayed on LCD.

2

Rain Forecast Mode

Fetches real-time rain probability from WeatherAPI using Wi-Fi connectivity. Displays forecast data to help decide whether irrigation is needed.

3

Hybrid Mode (Soil + Rain)

The smartest mode — combines soil moisture readings with rain probability. Only irrigates when soil is dry AND rain chance is below the threshold, preventing unnecessary watering.

4/5

Manual Override

Direct pump ON/OFF control for manual operation when automated control isn't needed.

Key Code — Hybrid Decision Logic

The core intelligence of the system lies in the hybrid mode, which combines soil moisture readings with rain probability to make smart irrigation decisions:

MicroPython main.py — mode_both() 
def mode_both():
    soil = read_soil()
    rain_prob = get_rain_probability()
    lcd.clear()
    lcd.putstr(f"Soil:{soil}")
    lcd.move_to(0,1)
    if rain_prob is None:
        lcd.putstr("Rain:Error   ")
        return
    lcd.putstr(f"Rain:{rain_prob}%")

    # Only irrigate if soil is DRY and rain chance is LOW
    if soil >= MOISTURE_THRESHOLD and rain_prob < RAIN_THRESHOLD:
        irrigate_once()
    else:
        lcd.putstr("Skip Irrigation")

The system uses a dual-threshold approach — irrigation only triggers when both conditions are met: soil moisture is below the threshold and rain probability is low. This prevents wasteful irrigation when rain is expected.

Challenges & Learnings

📡

Reliable Wi-Fi on Microcontroller

Managing Wi-Fi reconnection and API calls on the Pico W's limited memory required careful error handling and resource management.

🌱

Sensor Calibration

Capacitive soil moisture sensors needed careful calibration across different soil types and moisture levels to find accurate thresholds.

⚡

Relay Timing & Safety

Implementing active-low relay control with proper pump runtime limits to prevent motor burnout and water flooding.

🏗️

3D Printed Enclosure

Designed and 3D-printed a custom enclosure to house all electronics, with cutouts for the LCD, keypad, antenna, and sensor cables.
